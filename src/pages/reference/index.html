Kestrel Reference

Disable all links

Enabled by Default? No

Disables all links on the current page. Note: this command doesn't stop JavaScript-based navigation, so it may not work on all "links."

This command is not a toggle as it is made to lessen distractions.

Open settings page

Enabled by Default? No

Opens Kestrel's settings page

Refresh all tabs: soft

Enabled by Default? Yes

Aliases:

  • Refresh all tabs: hard

Refreshes all tabs in every window. If the "hard," variant is chosen, the cache is bypassed when refreshing the tabs.

Scroll to top of page

Enabled by Default? Yes

Aliases:

  • Scroll to 1/4 of the page
  • Scroll to middle of the page
  • Scroll to 3/4 of the page
  • Scroll to bottom of the page

Scrolls to the given distance in the currently active tab.

Toggle animations: off

Enabled by Default? Yes

Aliases:

  • Toggle animations: on (after animations are toggled off)

Turns the currently active tab's animations on or off.

Toggle page editing

Enabled by Default? No

Enables or disables editing the currently active page.

Toggle media

Enabled by Default? Yes

Aliases:

  • Toggle images
  • Toggle video

Shows or hides the selected media. If the "media," variant is chosen, then both videos and images are hidden.

Toggle minimap

NOTE: This command is currently non-functional.

Enabled by Default? No

Toggles an interactive minimap of the page.

Background

Background tasks aren't enabled by default.

Due to enabled tasks being chosen by the user, it is not always guaranteed that tasks that start at page load will be completed successfully. For example, the page loader will not always work if a site loads exceptionally fast, because it must first be injected.

These tasks may also slow down performance. Keep this in mind when choosing which tasks to enable.

Page Loader

Enables a loading bar at the top of each page, representing the loading status of the page. The colour can be customized in the Loading Bar Colour section of the Settings page.

Please note that the loader is not always guaranteed to work. Since it must be injected into a page first, it is not always able to begin once the page is first navigated to.
